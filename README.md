# 🔬 Financial Data Playground (2023 - 2025)

Este repositório reúne meus projetos desenvolvidos entre 2023 e 2025, focados em Economia Quantitativa, Engenharia de Dados e Automação. Utilizo Python e SQL para converter teorias complexas em soluções práticas, abrangendo desde a otimização de portfólios financeiros e gestão de risco até a criação de ferramentas de análise de dados para economias virtuais. 
---

## 🏛️ Estrutura Arquitetural do Repositório

O laboratório foi segmentado em dois super-focos de atuação madura, com ênfase no uso rigoroso do ecossistema assíncrono do Python (`Pandas`, `NumPy`, `YFinance`, e bibliotecas Científicas/Visuais).

### 1. Finanças Quantitativas (`/Financas_e_Quant`)
O módulo onde a Ciência de Dados cruza com Estratégias de Capital Institucional.\
Nesta pasta, o foco reside na execução inter-temporal, indo desde a segurança de retornos até a caçada por Alpha:

*   **`01_Gestao_Risco_e_Otimizacao_Portfolio.ipynb`**: *A Fundação Clássica*. Consolidação da exploração da covariância sobre séries temporais integradas com modelagem otimizada de Markowitz.
*   **`02_Backtesting_e_Estrategias_Quantitativas.ipynb`**: *A Esteira Algorítmica*. Resoluções do Itaú Quant Challenge focadas em testes rigorosos (`intraday tick-data` do Yahoo Finance) acionando estratégias de fluxo.
*   **`03_Processamento_Sinais_Fourier_Paparazi.ipynb`**: *A Exploração de Vanguarda*. O processamento analítico via Trigonometria Contínua (análogo à **Transformada de Fourier**) para extração preditiva de tangentes não lineares de ciclo de preço.

### 2. Tratamento e Visualização de Dados (`/Tratamento_de_dados`)
Onde dados complexos tomam forma estratégica para Tomada de Decisão (*Big Data Gov*).\
Nesta pasta modelamos as bases públicas do Seguro Rural Brasileiro para simular *Data Wrangling* no ambiente corporativo:

*   **`01_Limpeza_e_Agrupamento_Dados.ipynb`**: Focado no poder transformacional do *Data Prep*. Lidando com imputação de nulos (`dropna`, `fillna`), cast inferido, sanitização de strings, feature engineering pesada e redução de dimensionalidade por interações escalares (`groupby`, pivots).
*   **`02_Analise_Exploratoria_e_Estudo_Goias.ipynb`**: O coroamento visual dos dados (`EDA`). Uso avançado do ecossistema Matplotlib/Seaborn para comprovar (ou destruir) percepções errôneas do negócio. Abordamos dispersões de valores de apólices e isolamos macro-regiões no Cerrado a nível cirúrgico.

### 3. Automação e Backend (`/Projetos_Automacao_Backend`)
Soluções de engenharia de software focadas em extração de dados em larga escala, dashboards interativos e visão computacional:

*   **`PaxDei_Market_App`**: Uma ferramenta robusta de análise de mercado para o MMO Pax Dei. Inclui uma esteira ETL completa (Python/HuggingFace), um servidor backend em Flask/Python e um dashboard interativo para identificação de oportunidades de arbitragem e crafting.
*   **`Sucram_Video_Scene_Finder`**: Aplicação de Visão Computacional (OpenCV) e IA para detecção automática de cenas em vídeos. Inclui uma interface gráfica (GUI), rotinas de download automatizado e integração com Google Drive para processamento de ativos de mídia.

## 🚀 Como Explorar e Executar

Sinta-se à vontade para navegar diretamente pelo GitHub. Quase a sua totalidade visual (como gráficos complexos, dezenas de painéis EDA de Goiás e os arrays do Paparazi) mantiveram seus estados consolidados, resultando em leitura direta e limpa!

Caso queira clonar localmente:

```bash
git clone https://github.com/NkNaite/Trabalhos-2023-2025-.git
cd Trabalhos-2023-2025-
pip install -r requirements.txt
```

> **Atenção técnica para os diretórios Quant:** Se for rodar a malha de algoritmos em sua máquina, prefira executá-los em sequência modular ou limite as extrações em massa da API do YFinance!
